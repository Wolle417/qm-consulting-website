- Aber: Aufwändigste Entwicklung (3-4 Wochen)

### Kritische Erkenntnisse über das Konzept

#### 1. Der Pharma-Gap ist real
Thomas' Feedback ist eindeutig: Wenn die Tools nur MedTech/ISO 13485 sprechen, verliert QCore 40% der Zielgruppe. Lösung: Jedes Tool braucht einen sichtbaren Modus-Schalter (MedTech | Pharma) — wie der NC-Triage ihn schon hat.

#### 2. Das Readiness-Check-Problem
Lena braucht den Readiness Check DRINGEND — aber Stefan sagt, er ist noch nicht gut genug. Das ist ein Problem, weil der Readiness Check für Einsteiger der natürliche Einstiegspunkt wäre. **Empfehlung: Readiness Check als Priorität NACH dem Sample Size Calculator verbessern, bevor weitere neue Tools gebaut werden.**

#### 3. Bundle-Previews fehlen
Lena (und vermutlich viele andere) kaufen keine Templates "blind". Zeigt ein Beispiel-Dokument oder zumindest Screenshots der enthaltenen SOPs. Das ist kein neues Tool, sondern eine Optimierung der bestehenden Produktseiten — und könnte die Conversion stärker steigern als ein neues Tool.

#### 4. Die Berater-Multiplikatoren
Markus' Hinweis ist strategisch wertvoll: Ein Berater empfiehlt an 8-10 Kunden. 50 Berater = 500 QMBs. Ideen: LinkedIn-Beiträge gezielt an QM-Berater richten. "Empfehlen Sie Ihren Kunden dieses Tool" als Framing.

#### 5. Daten-Durchlässigkeit zwischen Tools
Sabines Wunsch, dass NC-Triage-Ergebnisse automatisch ins Cockpit fließen, ist der Schlüssel zum Ökosystem-Gedanken. Technisch über localStorage-Keys lösbar, wenn alle Tools im gleichen Domain-Kontext laufen.
